Chat Oldtimes — Old-School WebSocket Chat

A minimal React + Vite frontend paired with a Go 1.26 backend. Real-time WebSocket messaging, clean architecture, and decoupled design—perfect for learning concurrent patterns, message dispatch, and composable frontend-backend systems.

Quickstart API

Features

  • Real-time Chat: WebSocket-based messaging between connected clients.
  • React + Vite: Modern frontend with hot module replacement.
  • Go 1.26 Backend: Goroutines, channels, and solid concurrency patterns.
  • Room Support: Join rooms, broadcast messages across clients.
  • WebSocket Client: Helper in wsChat.ts.

Built-in

  • Concurrency: Go backend leverages goroutines and channels.
  • Decoupled: Frontend and backend are independent.
  • Business Logic: Backend applies rules for room and message handling.
  • Composable: Easy to pair with other services or extend with persistence.

▶️ Quick Run (Local) #

Prerequisites #

  • Go 1.26+ installed
  • Node.js 18+ and npm installed
  • Backend and frontend cloned in separate directories

Backend Setup #

# Navigate to the backend directory
cd /path/to/chat-oldtimes-backend

# Run the server
go run .

# Server listens on :8080

Frontend Setup #

# Navigate to the frontend directory
cd /path/to/chat-oldtimes

# Install dependencies
npm install

# Start dev server
npm run dev

# Open http://localhost:5173

🔌 WebSocket API #

Connect #

ws://localhost:8080/ws?nick=YOUR_NICK

Replace YOUR_NICK with your desired username.

Join Room #

{"type":"join","room":"geral"}

Send Message #

{"type":"message","text":"hello everyone"}

Change Nickname #

{"type":"nick","nick":"newNickname"}

🏗️ Architecture #

Backend (Go) #

  • Message Types: Defines protocol messages (join, message, nick, etc.).
  • Room Manager: Manages active rooms and client connections.
  • Goroutines & Channels: Concurrent message dispatch and client handling.
  • Broadcasting: Sends messages to all clients in a room.

Frontend (React + Vite) #

  • wsChat.ts: WebSocket client helper with connect/send/disconnect.
  • App.tsx: Main UI component with room and message state.
  • Styles: Clean, minimal dark theme in CSS.

⚡ Why It Matters #

  • Educational Value: Learn WebSocket mechanics on client and server.
  • Practical Patterns: Go concurrency with goroutines/channels, React hooks.
  • Decoupled Architecture: Frontend is independent—pair with other WebSocket servers.
  • Lightweight: Fast to spin up, great for prototypes and learning.
  • Real-time Foundation: Base for chat, notifications, collaborative tools.

🔄 Example Flow #

  1. Client connects: ws://localhost:8080/ws?nick=Alice
  2. Backend creates connection: Goroutine spawns for this client.
  3. Client joins room: Sends {"type":"join","room":"geral"}
  4. Backend adds to room: Notifies other clients in room.
  5. Client sends message: {"type":"message","text":"Hey!"}
  6. Backend broadcasts: Delivers message to all clients in room.
  7. UI updates: React state updates, message appears in chat.

🔧 Troubleshooting #

Connection Refused #

Issue: Frontend cannot connect to backend.
Solution: Ensure backend is running on :8080. Check firewall and CORS settings. Try curl ws://localhost:8080/ws from terminal.

Hot Reload Not Working #

Issue: Vite hot reload not triggering.
Solution: Restart npm run dev. Check that file watchers are enabled. On Linux, may need to increase file descriptor limits.

WebSocket Closes Immediately #

Issue: Connection established then closes.
Solution: Check backend logs for errors. Verify message format is JSON. Ensure server-side error handling logs the issue.
